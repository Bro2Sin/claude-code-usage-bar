# claude-code-usage-bar (личный форк)

Статус-строка для Claude Code — показывает расход лимитов (5 часов / 7 дней) и заполнение контекстного окна прямо над полем ввода, в виде цветных заливающихся баров.

Это форк оригинального проекта [leeguooooo/claude-code-usage-bar](https://github.com/leeguooooo/claude-code-usage-bar) с тремя личными правками (см. ниже). Вся остальная документация, список всех фич и настроек — в оригинале, эта версия README короче и написана под мой собственный сценарий использования.

## Как выглядит

```
5h[   18%    ]3h29m →62% | 7d[    4%    ]5d13h →27% | Sonnet 5[   27%    ](270.0k/1.0M)
⤷ projects (no git) · +396 -47 · v3.29.11
```

| Часть | Что значит |
|---|---|
| `5h[18%]3h29m` | Расход 5-часового лимита (18%), через сколько сбросится (3ч29м) |
| `7d[4%]5d13h` | Расход недельного (7-дневного) лимита, время до сброса |
| `→62%` / `→27%` | Прогноз — если тратить в том же темпе, столько процентов будет израсходовано к моменту сброса. Это просто справочная цифра, на цвет бара **не влияет** (см. правки ниже) |
| `Sonnet 5[27%](270.0k/1.0M)` | Модель, заполнение контекстного окна (27%), сколько токенов из скольки использовано |
| `⤷ projects (no git) · +396 -47` | Название рабочей папки, git-статус (тут его нет), сколько строк добавлено/удалено за сессию |

Цвета баров: зелёный → жёлтый → красный, по порогам 30%/70% (для 5h/7d) и 70%/85% (для контекста).

## Мои правки поверх оригинала

1. **Цвет бара 5h/7d считается по факту, не по прогнозу.** В оригинале бар мог стать красным при 15% факта, если прогноз предсказывал добор до 100% к сбросу — сбивало с толку. Теперь цвет = текущий процент, прогноз просто печатается рядом текстом (`→62%`), не красится.
2. **Убран смайлик-часы (⏰) перед временем до сброса.** Было `⏰3h29m`, стало просто `3h29m`.
3. **Добавлен бар для контекстного окна.** В оригинале контекст был просто текстом `Sonnet 5(270k/1M)`. Теперь это заливающийся бар с цветом: `Sonnet 5[27%](270.0k/1.0M)`.

Правки — в трёх файлах: `src/claude_statusbar/core.py`, `src/claude_statusbar/progress.py`, `src/claude_statusbar/styles.py`. Полный дифф — в коммите [`6138fa6`](https://github.com/Bro2Sin/claude-code-usage-bar/commit/6138fa6).

## Установка (на новой машине)

Через [uv](https://docs.astral.sh/uv/) (ставит без root, в изолированное окружение):

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source $HOME/.local/bin/env
uv tool install git+https://github.com/Bro2Sin/claude-code-usage-bar.git
cs --setup
```

Перезапустить Claude Code — бар появится сам.

## Обновление

Если в оригинальном проекте выйдут новые фичи и захочется их подтянуть — можно смёржить апстрим в свой форк:

```bash
cd claude-code-usage-bar
git remote add upstream https://github.com/leeguooooo/claude-code-usage-bar.git   # один раз
git fetch upstream
git merge upstream/main   # если будут конфликты в трёх патченых файлах — решать вручную
git push origin main
```

Переустановить локально после этого:

```bash
uv tool install --force git+https://github.com/Bro2Sin/claude-code-usage-bar.git
cs --setup
```

## Настройки

Все команды `cs config set ...`, стили (`classic`/`capsule`/`hairline`), темы и остальные фичи — без изменений, подробности в [оригинальном README](https://github.com/leeguooooo/claude-code-usage-bar#readme).

Текущие настройки этого форка:

```bash
cs config show
```
